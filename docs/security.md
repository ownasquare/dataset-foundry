# Security and privacy model

Dataset Foundry is local-first, but imported training data and provider prompts can still be
sensitive. Operators should treat seeds, candidates, review notes, and exports as customer data.

## Defaults

- The server binds to `127.0.0.1` by default.
- The default provider is offline and makes no network calls.
- Live-provider runs require explicit external-data-transfer approval.
- Provider credentials stay in server environment variables and never appear in API responses,
  browser storage, generated manifests, logs, or exports.
- Upload size, row count, extension, and canonical schema are bounded.
- Artifact paths are generated by the server beneath one configured directory.

## Network exposure

Loopback is suitable for a single-user local installation. If the API is intentionally bound to a
non-loopback interface, configure the application API key, put the service behind a TLS-terminating
reverse proxy, restrict ingress, and use a production database/queue appropriate to the deployment.
The proxy must inject `X-API-Key` for `/api/v1` and `/metrics`; the browser-facing SPA and static
assets remain public so they can load before the proxy authenticates data requests. The bundled
Compose file explicitly opts into unauthenticated container binding only because it publishes the
service to host loopback (`127.0.0.1:8765`). Do not reuse that opt-in with a public port mapping.

## Provider transfer

The preflight response identifies whether a provider is live and requires external data transfer.
The run request must affirm that transfer. Dataset Foundry does not inspect or redact arbitrary
proprietary content automatically; dataset owners are responsible for data-classification and
provider-contract decisions before approving transfer.

## Logging and evidence

Operational logs may include generated IDs, counts, state changes, provider/model names, latency,
and request IDs. They must not include credential values or complete prompt/seed bodies. Provider
refusals and errors are recorded as classifications and sanitized messages.

Export manifests include provenance and hashes, not provider keys. Hashes prove artifact integrity;
they are not a substitute for access control or encryption at rest.

## Container posture

The bundled image runs as a non-root user. Compose drops Linux capabilities, prevents privilege
escalation, uses a read-only root filesystem with a bounded temporary filesystem, and mounts a
dedicated writable data volume. Secrets must be injected at runtime rather than built into the
image or committed to Compose.
